# 2026-04-14 MNIST手写数字识别 v1.0

## 🔍 今日任务
使用 PyTorch 构建 **2层全连接神经网络** 识别 MNIST 手写数字，重点攻克 `torch.nn` 模块化封装与 `ReLU` 激活函数的底层原理。

## 💥 遇到的问题 & 解决方案

| 问题 | 根本原因 | 解决方案 | 收获 |
| :--- | :--- | :--- | :--- |
| **线性堆叠无效** | 多层 `Linear` 若无激活函数，数学上等价于单层线性变换 | 插入 `nn.ReLU()`<br>打破线性，引入非线性特征 | **模型能力本质**：<br>深度学习的“深度”价值源于非线性变换 |
| **梯度衰减风险** | 链式法则导致连乘效应（如 Sigmoid 导数<1，连乘后趋近0） | 选用 **ReLU** 激活函数<br>正区间导数恒为1，梯度无损传播 | **工程权衡**：<br>ReLU 是计算效率与梯度稳定性的最佳平衡点 |
| **代码冗余难管** | 手动管理权重矩阵 (W, b) 繁琐且易错 | 使用 **`torch.nn`** 模块<br>自动注册参数与生命周期管理 | **框架思维**：<br>利用 `nn.Module` 像搭乐高一样构建网络 |

## 📊 实验数据

| 参数配置 | 理论表现 | 关键发现 |
| :--- | :--- | :--- |
| **Linear Only** (无激活) | 约 92.0% | **模型退化**：无论加多少层，效果等同于单层感知机 |
| **Linear + Sigmoid** | 收敛极慢 | **梯度消失**：深层网络参数几乎不更新，训练停滞 |
| **Linear + ReLU (v1.0)** | **收敛快** | **最佳实践**：计算仅需 `max(0,x)`，且有效保留梯度信号 |
